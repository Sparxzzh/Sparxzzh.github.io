Object通用方法：

1.equals()

  • 满足自等传递一致对称
  • 与 == 区别在于，equals实现判断对象引用的值域是否相等，前者判断引用对象是否相等 前为后的充分不必要

    2.hashcode()

  • 计算哈希值，覆盖equals()可实现哈希值相等，存储在hashset中的对象也一致
  • 保证均匀分布，考虑各值域表示为R进制数，并计算，为保证不乘法溢出,R选用奇素数31

    3.toString()

  • 对象调用一般返回 ToStringExample@4554617c这种形式的无符号散列码

    4.clone()

  • protect类型，其他类调用时需显示重写该方法，并实现cloneable接口规定，否则抛出异常。
  • 浅拷贝：拷贝对象和原对象的引用类型引用相同对象。
  • 深拷贝:重写clone方法时，在方法内创建了一个新的对象引用并实现自定义拷贝(如数组)，拷贝对象引用类型引用的是一个新对象。

最好不要用clone而改为自定义cloneConstructor的拷贝构造函数或拷贝方法避免复杂的异常处理

访问权限

  • 类可见：其他类操作这个类创建对象实例。
  • 成员可见：其他类可操作这个类的对象实例访问该成员。
  • 信息封装：隐藏细节用API通信，尽可能的使成员不被外界访问。
  • 里氏替换：子类重写要符合该原则，即重写实例优先级高与父类实例
  • 对象调用和对象方法调用字段：如声明公有字段，对象可随意调用，失去控制，应声明私有字段让对象方法调用。包级私有类和嵌套类除外。

抽象类和接口

  • 抽象类：配合抽象方法使用，不能实例化，实例化交给继承的子类，起到规范化开发的作用。
  • 接口：抽象类的扩展，java8之前不能有默认实现，由于维护成本过高(修改频繁)，改为可有默认实现，和抽象类一样可用implement 实现方法重写。
  • 比较：设计层面。抽象类子类对象必须替换所有父类的is-a 关系，接口则为like - A 的非严格要求；使用层面。一个类可实现多个接口，但不能继承多个抽象类；接口字段为static和final 成员为public ，抽象类无限制；

使用选择

  • 使用接口：不相干类实现同一方法，多重继承
  • 使用抽象类：开发规范，相关类共享代码，控制继承成员的访问权限；继承非静态和非常量字段

super

  • 访问父类构造函数，特别是非默认构造函数
  • 重写的父类方法可引用父类方法实现；

重写

  • 用父类完全相同的声明实现的一个方法。
  • 重写用@override检查是否满足权限、返回类型、异常类型的限制。
  • 方法调用为先检查调用自身方法，在检查父类，最后可对继承的对象参数转型。

重载

  • 同一个类中，参数类型、顺序、个数至少一个不同。
  • 注意返回值不同，其余相同不为重载。

概述

why git ?

首先，代码开发过程中对代码进行回滚和备份的操作很重要，其次，代码存储设备难免存在丢失的情况，这使得我们需要对代码进行远端存储，基于git操作开发的github给我们提供了便利。最重要的原因是，代码的版本控制在团队中能够极大的提高工作效率，尤其是在大公司得以体现。

git 简介

git 是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。
分布式相比于集中式的最大区别在于开发者可以提交到本地，每个开发者通过克隆（git clone），在本地机器上拷贝一个完整的Git仓库。

git 三大阶段 ：1.修改 2.暂存 3.提交

  • git add : 操作文件

  • git commit -m “内容记录” ：呈交 简单粗暴方式

  • git diff : 比较改变内容

  • git diff –cached : 捕捉到暂存内容

  • git status : 查看各文件追踪状态

  • git commit + vim :比较正式的呈交

  • git commit -a -m “内容记录” ：跳过暂存和消息输入 直接提交
